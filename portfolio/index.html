Data Strategy for a Hospital Chain

This project comprised the conception of a data strategy for a US based hospital chain operating across 3 states. We followed a tailored TOGAF framework for designing the enterprise data strategy. The data strategy included current state analysis, identification of issues and goals through stakeholder interviews, proposing a future state, and drawing a transformation road-map, staying mindful of security, governance, data management, analytics and reporting requirements, as well as other organizational needs.

My role in this project was quite versatile. I led several phases of the project including the stakeholder interviews for identifying the current state problems, as well as in drawing up the future state architecture most suited to eradicate those problems and help achieve the organizational goals. I was also part of the team which put together a 2 year transformation roadmap to get to the future state.

Technology stack involved overall was Azure Cloud Ecosystem, Epic Health Management System, SAP HANA, Hadoop, Oracle, Informatica as well as Denodo
